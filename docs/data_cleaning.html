Limpieza y Preparación de Datos

Procesamiento de tratamiento de Datos

En esta sección se documenta el proceso de adquisición de datos hasta la transformación del dataset original a un formato apto para el análisis estadístico y elaboración de visualizaciones técnicas.

El objetivo es garantizar que los datos estén limpios, estructurados y listos para responder a las preguntas planteadas mediante análisis visual.

Proceso ETL

Los datos están almacenados en un archivo .csv por lo cual su ingesta se limita a leerlo y transformador a un DataFrame de pandas.

El dataset original presenta un formato “ancho” con una columna por cada hora del día, lo cual dificultaría el análisis estadístico. Se aplica una transformación a formato “largo” para facilitar su análisis. Para esto se utiliza la función melt de pandas, que consolida las 24 columnas de horas en una sola variable categórica llamada Hora y una variable cuantitativa llamada Generacion_MW.

El dataset original contiene el caracter - ante ausencia de generación; este es reemplazado por el valor numérico 0 para asegurar que las visualizaciones reflejen correctamente los periodos de generación nula.

1. Carga del Dataset

Cargamos los datos utilizando Pandas, gestionando los separadores de miles (punto) y decimales (coma) detectados en el archivo original.

# Función para limpieza de datos
def limpiar_texto(texto):
    if pd.isna(texto) or not isinstance(texto, str):
        return texto
    # Normalizar (separa la letra del acento) y eliminar acentos
    texto = unicodedata.normalize('NFD', texto)
    texto = "".join([c for c in texto if unicodedata.category(c) != 'Mn'])
    # Mayúsculas y eliminar espacios extra
    return texto.upper().strip()

2. Transformación a Formato ancho, limpieza de datos e impugación de valores

Utilizamos la función melt para consolidar las 24 columnas de horas en una sola variable categórica llamada Hora y una variable cuantitativa llamada Generacion_MW. Originalmente el dataset contiene el caracter - ante ausencia de generación; este es reemplazado por el valor numérico 0.

Se identifica si existen filas con fechas duplicadas:

Code 
# Aseguramos que 'Hora 1' sea numérica (evita errores de tipo 'object')
df_raw['Hora 1'] = pd.to_numeric(df_raw['Hora 1'], errors='coerce')

# Limpiar etiquetas de COLUMNAS
df_raw.columns = [limpiar_texto(col) for col in df_raw.columns]

# Actualizo el nombre de la columna año
df_raw = df_raw.rename(columns={'ANO': 'ANIO', 'DIA':'FECHA' })

# Limpiar contenido de la columna UNIDAD
# Nota: Ahora la columna se llama 'UNIDAD' en mayúsculas por el paso anterior
df_raw['UNIDAD'] = df_raw['UNIDAD'].apply(limpiar_texto)

# Consolidación (Melt) con los nombres ya limpios
id_vars = ['UNIDAD', 'ANIO', 'FECHA'] # Usamos los nombres ya normalizados
hora_cols = [f'HORA {i}' for i in range(1, 25)]

df_consolidado = df_raw.melt(
    id_vars=id_vars,
    value_vars=hora_cols,
    var_name='HORA',
    value_name='GENERACION_MW'
)

# Limpieza final de la variable HORA para que sea solo el número
df_consolidado['HORA'] = (
    df_consolidado['HORA']
    .str.extract(r'(\d+)')  # r indica raw string, más seguro para regex
    .fillna(0)              # Si no encuentra número, pone 0
    .astype(int)
)

# Limpieza de valores: los guiones '-' se interpretan como ausencia de generación (0)
df_consolidado['GENERACION_MW'] = (
    df_consolidado['GENERACION_MW']
    .replace('-', '0')
    .pipe(pd.to_numeric, errors='coerce') # Convierte a float y deja NaN si hay error
    .fillna(0)                            # Convierte los NaN en 0
)

# Aseguramos que DIA sea tipo datetime
df_consolidado['FECHA'] = pd.to_datetime(df_consolidado['FECHA'])
# Creamos la columna FECHA_HORA
# Sumamos a la fecha original una cantidad de horas proporcional al valor de 'HORA'
df_consolidado['FECHA_HORA'] = df_consolidado['FECHA'] + pd.to_timedelta(df_consolidado['HORA'], unit='h')

# Elimino errores por asignación de la hora 24 (solapamiento por hora 0 del día siguiente)
df_consolidado['FECHA_HORA'] = np.where(
    df_consolidado['HORA'] == 24,
    df_consolidado['FECHA_HORA'] - pd.Timedelta(seconds=1),
    df_consolidado['FECHA_HORA']
)

# identifico valores duplicados
duplicados = df_consolidado[df_consolidado.duplicated(subset=['UNIDAD', 'FECHA_HORA'], keep=False)]
print(f"Se encontraron {len(duplicados)} filas en conflicto.")

# Definimos el orden deseado de las columnas
columnas_ordenadas = ['FECHA_HORA', 'FECHA', 'ANIO', 'HORA', 'UNIDAD', 'GENERACION_MW']

# Reorganizamos el DataFrame
df_consolidado = df_consolidado[columnas_ordenadas]

Justificación de decisiones

  1. Uso de Melt: Altair requiere que los datos estén en formato largo para poder mapear canales visuales (como el color o el eje X) a columnas específicas del dataframe.
  2. Manejo de Nulos: Se sustituyeron los guiones y valores nulos por 0 para garantizar que los gráficos de áreas y heatmaps no presenten “huecos” que distorsionen la interpretación técnica de la disponibilidad.
  3. Tipado Explícito: Al definir la generación como cuantitativa (Q) y la fecha como temporal (T), permitimos que las herramientas de visualización realicen agregaciones automáticas como promedios o sumas por mes
